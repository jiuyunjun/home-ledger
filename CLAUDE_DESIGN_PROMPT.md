# CLAUDE_DESIGN_PROMPT.md

Copy the following prompt into Claude Design.

---

你是一个资深产品设计师和前端 UI 设计师。请为一个“家庭记账工具”的 Web MVP 设计完整网页界面。目标是先做网页，之后再做 Android App，所以设计必须 mobile-first、responsive，并且组件结构清晰，方便以后复用。

## 产品背景

这是一个给我和女朋友使用的私人家庭记账工具。生活场景以日本为主，默认主要使用日元 JPY，但也需要支持人民币账户 CNY/RMB。用户需要可以快速记录支出、入账和账户转换，也可以上传小票图片，让 AI 自动识别内容并分类。但 AI 结果必须由用户确认和修改后才能正式入账。

## 核心角色

系统有三个记账角色：

1. 我
2. 女朋友
3. 家庭共通

浏览器需要记住当前选择的角色。界面上要有明显的角色切换入口。新增支出或入账时默认使用当前角色，但用户可以修改。

## 账户和支付方式

需要支持：

- 现金
- PayPay
- 多张信用卡
- 日本银行账户
- 人民币账户 / CNY / RMB
- 未来可能增加其他账户或支付方式

## 交易类型

需要支持三种交易：

1. 支出：买东西、吃饭、交通、房租等。
2. 入账：工资、奖金、退款、报销、利息等。
3. 账户转换：日元银行账户转人民币账户、银行账户还信用卡、现金存入银行等。

注意：入账不是负数支出，账户转换不是支出。

## 必须设计的页面

### 1. Dashboard 首页

展示：
- 本月总支出
- 我的支出
- 女朋友支出
- 家庭共通支出
- 本月入账摘要
- 各账户余额摘要，包括人民币账户
- 按分类的支出摘要
- 本月预算使用情况，突出接近上限或超出上限的分类
- 最近交易列表，区分支出、入账、转换
- “手动记账”、“入账”、“账户转换”和“小票上传”主要按钮

### 2. 手动记账页面

需要支持三种模式：支出、入账、账户转换。

支出字段：
- 角色
- 日期
- 金额
- 币种：JPY / CNY
- 分类
- 支付方式 / 支出账户
- 店铺 / 标题
- 备注

入账字段：
- 角色
- 日期
- 金额
- 币种：JPY / CNY
- 入账分类，例如工资、奖金、退款、报销
- 入账账户
- 来源 / 标题
- 备注

账户转换字段：
- 日期
- 转出账户
- 转入账户
- 转出金额
- 转入金额
- 汇率，可选
- 手续费，可选
- 备注

要求：
- 手机端输入要快
- 金额输入要突出
- 常用分类、账户和支付方式可以快速选择
- 转账不应该显示为支出

### 3. 小票上传页面

功能：
- 支持上传单张或多张小票图片
- 支持拖拽上传
- 手机端支持拍照上传
- 上传时可以输入任意文字备注，作为 AI 分类提示。例如：“这张算餐饮”、“这是公司报销”、“不要按商品拆分”、“这是人民币支付”
- 上传后显示每张图片的预览
- 显示 AI 识别状态：等待中、识别中、需要确认、失败

### 4. AI 识别确认页面

这是最重要的页面。

要求：
- 左侧或上方显示小票图片预览
- 右侧或下方显示 AI 抽出的交易候选
- 显示用户上传时输入的 AI 提示备注
- 用户可以编辑：交易类型、日期、金额、币种、分类、支付方式、账户、角色、店铺名、备注
- 显示 AI 置信度
- 对不确定字段显示 warning
- 明确显示“确认入账”和“拒绝”按钮
- 不允许设计成 AI 自动入账
- 多张小票时要可以逐张确认

### 5. 明细页面

功能：
- 交易列表
- 按月份筛选
- 按交易类型筛选：支出 / 入账 / 转换
- 按角色筛选
- 按分类筛选
- 按账户或支付方式筛选
- 搜索店铺名、来源或备注
- 编辑和删除交易

### 6. 每月预算页面

功能：
- 可以设置每月某个类别的支出上限
- 字段：月份、角色范围、分类、预算金额、币种、提醒阈值、是否启用
- 展示已使用金额、剩余金额、使用百分比
- 接近上限时显示黄色 warning
- 超过上限时显示红色 warning
- 预算只统计支出，不统计入账和账户转换

### 7. 固定支出 / 固定入账页面

功能：
- 设置每月固定支出，例如房租、保险
- 设置每月固定入账，例如工资
- 字段：交易类型、名称、金额、角色、分类、账户 / 支付方式、每月几号、开始日期、是否启用
- 展示下次生成日期
- 可以暂停或编辑

### 8. 设置页面

功能：
- 角色管理
- 账户管理，包括人民币账户
- 支付方式管理
- 分类管理，包括支出分类和入账分类
- 当前浏览器默认角色
- 数据导出入口，标注 UTF-8

## 设计风格

请使用：
- 简洁、现代、干净
- 适合私人财务工具，不要太花哨
- 色彩温和，有安全感
- 手机端优先
- 信息层级清楚
- 卡片式布局
- 适合中文界面
- 可以带少量日式生活感，但不要做成可爱风

## 输出要求

请输出：
1. 完整页面结构
2. 每个页面的 UI layout
3. 组件列表
4. 关键交互说明
5. Responsive 设计说明
6. 可以直接给前端实现的 React + Tailwind 风格组件命名建议
7. 使用 mock data 展示界面

## 重要约束

- AI 识别结果必须是“待确认”的 draft。
- 用户确认后才正式入账。
- 浏览器记住当前角色，但后端仍需要验证权限。
- JPY 金额用整数日元。
- CNY 金额显示到分，但内部避免浮点误差。
- 入账不是负数支出，账户转换不是支出。
- 每月预算只统计正式确认的支出。
- 文件读写和导出都使用 UTF-8。
- 不要设计复杂企业财务系统，只做两个人家庭使用的轻量工具。
